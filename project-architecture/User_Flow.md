# User Flow

## Overview

The **Book a Doctor** application follows a structured workflow that allows patients, doctors, and administrators to perform their respective tasks efficiently. The workflow includes user registration, login, doctor browsing, appointment booking, doctor application, appointment approval, and appointment history management.

---

## 1. Registration Flow

* The user opens the Book a Doctor application.
* The user selects **Register** from the landing page.
* The user chooses the appropriate role (User or Admin).
* The user enters the required registration details.
* After successful registration, the user can log in to the application.

---

## 2. Login Flow

* The registered user enters their email address and password.
* The system validates the credentials.
* After successful authentication:

  * User is redirected to the **User Dashboard**.
  * Admin is redirected to the **Admin Dashboard**.

---

## 3. User Dashboard Flow

The User Dashboard allows the user to:

* View approved doctors.
* Search doctors by specialization and availability.
* Book appointments.
* Select appointment date and time.
* Upload medical documents.
* Apply as a doctor.
* View notifications.
* View appointment history.

---

## 4. Apply as Doctor Flow

* The user selects **Apply as Doctor**.
* The application form is completed.
* Professional details are submitted.
* The request is sent to the administrator for verification.

---

## 5. Admin Dashboard Flow

The administrator can:

* View doctor applications.
* Approve or reject doctor requests.
* View all registered users.
* View all registered doctors.
* Manage platform operations.

---

## 6. Appointment Booking & Approval Flow

* The user selects a doctor.
* The user books an appointment.
* The doctor receives the appointment request.
* The doctor reviews the request.
* The doctor approves or rejects the appointment.
* The appointment status is updated.
* The user receives a notification regarding the appointment.

---

## 7. Appointment History

### User

* View upcoming appointments.
* View completed appointments.
* Track appointment status.

### Doctor

* View assigned appointments.
* View completed consultations.
* Update appointment status.

---

## Conclusion

The Book a Doctor application provides a clear and efficient user workflow that enables secure registration, doctor management, appointment booking, approval, and appointment tracking. The structured flow improves communication between patients, doctors, and administrators while ensuring an organized healthcare appointment management system.

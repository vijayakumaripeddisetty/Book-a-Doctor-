# Creating Project Folder

## Objective

Create the initial project structure for the **Book a Doctor App** by organizing separate folders for the frontend and backend development.

## Steps

1. Create a new folder named **Book-a-Doctor-App**.
2. Open the folder in **Visual Studio Code**.
3. Inside the project folder, create two subfolders:
   - **client**
   - **server**
4. Verify that the project structure matches the following:

```
Book-a-Doctor-App/
│
├── client/
│
└── server/
```

## Folder Description

### client/

The **client** folder contains the frontend of the application. It is responsible for the user interface where patients, doctors, and administrators interact with the system. This includes features such as:

- User Registration and Login
- Doctor Browsing
- Appointment Booking
- Appointment History
- Document Upload
- Notifications
- Dashboard Interface

### server/

The **server** folder contains the backend of the application. It manages business logic, APIs, authentication, and database communication. This includes:

- REST API Development
- User Authentication and Authorization
- Doctor Profile Management
- Appointment Management
- Database Operations
- File Upload Handling
- Notification Services

## Project Structure

```
Book-a-Doctor-App/
│
├── client/
│   └── Frontend Source Code
│
└── server/
    └── Backend Source Code
```

## Outcome

After completing these steps, the project will have a well-organized folder structure with separate directories for the frontend and backend. This modular organization improves code maintainability, scalability, and team collaboration throughout the development of the **Book a Doctor App**.
